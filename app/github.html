Guía GitHub ADSO

1. Estructura del proyecto Organización de archivos

El proyecto tiene una estructura organizada para un sitio web de varias páginas:

taller3/
├── app/
│   ├── formularios.html
│   ├── tablas.html
│   └── repositorio.html
├── assets/
│   ├── css/
│   ├── img/
│   └── js/
├── index.html
└── README.md
2. Requisitos iniciales Herramientas necesarias
  • • Tener una cuenta activa en GitHub.
  • • Tener instalado Git en el computador.
  • • Tener instalado Visual Studio Code.
  • • Abrir la carpeta del proyecto desde Visual Studio Code.
3. Configuración de Git desde VS Code Pasos básicos

Abre la terminal integrada de Visual Studio Code

Menú Terminal → Nueva Terminal
Configura tu nombre y correo
git config --global user.name "Tu nombre completo"
git config --global user.email "tu_correo@ejemplo.com"
Verifica la configuración
git config --global --list
4. Inicializar Git en el proyecto Comandos básicos

Ubícate en la carpeta del proyecto desde la terminal de VS Code

git init
git add .
git commit -m "Primer commit: estructura inicial del proyecto"
Este primer commit guarda la versión inicial del proyecto, incluyendo todos los archivos creados hasta el momento.
5. Crear repositorio en GitHub
  1. Ingresar a la página oficial de GitHub.
  2. Hacer clic en el botón New repository.
  3. Asignar un nombre al repositorio, por ejemplo: taller3-bootstrap-adso.
  4. Seleccionar si será público o privado según tu preferencia.
  5. Crear el repositorio sin agregar archivo README si el proyecto ya tiene archivos locales.
